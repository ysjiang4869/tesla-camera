# Tesla Camera - 环境准备说明

## 项目概述

Tesla Camera 是一个特斯拉行车记录仪播放器，支持 Windows、macOS 和 Web 浏览器（Chrome 86+）平台。

## 环境要求

- **Node.js** - 推荐使用 LTS 版本
- **pnpm >=8.0.0** - 包管理器（本项目强制要求）
- **Rust** - 仅桌面应用开发需要

## 环境准备步骤

### 1. 安装 pnpm

```bash
npm install -g pnpm
```

验证安装：
```bash
pnpm --version
```

### 2. 安装 Rust（可选）

如果只需要运行 Web 版本，可以跳过此步骤。

如果需要开发桌面应用（Tauri），需要安装 Rust 工具链：

**macOS/Linux:**
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

**Windows:**
访问 [rustup.rs](https://rustup.rs/) 下载安装程序

验证安装：
```bash
rustc --version
```

### 3. 安装项目依赖

```bash
cd /Users/yuesong/Code/tesla-camera
pnpm install
```

### 4. 下载 FFmpeg 二进制文件（可选）

此步骤仅用于**桌面应用**的视频导出功能。Web 版本不需要。

**方法一：自动下载**
```bash
./init-binaries.sh
```

**方法二：手动下载**

如果网络连接 GitHub 失败，可以手动下载：
1. 访问 https://github.com/Mario34/tesla-camera/releases/download/v1.0.0-pre/ffmpeg.zip
2. 解压到 `src-tauri/binaries/` 目录
3. 确保以下文件存在：
   - `src-tauri/binaries/ffmpeg` (macOS/Linux)
   - `src-tauri/binaries/ffmpeg.exe` (Windows)
   - `src-tauri/binaries/ffmpeg-aarch64-apple-darwin` (Apple Silicon Mac)
   - `src-tauri/binaries/ffmpeg-x86_64-apple-darwin` (Intel Mac)
   - `src-tauri/binaries/ffmpeg-x86_64-pc-windows-msvc.exe` (Windows)

## 故障排查

### 1. pnpm 版本过低

```bash
pnpm install -g pnpm@latest
```

### 2. FFmpeg 下载失败

网络问题导致无法从 GitHub 下载，可以：
- 使用代理
- 从 Gitee 镜像下载
- 跳过此步骤（仅影响桌面应用的导出功能）

### 3. Tauri 构建失败

确保：
- Rust 工具链已正确安装
- Xcode 命令行工具已安装（macOS）
- FFmpeg 二进制文件已下载

### 4. 端口占用

开发服务器默认使用 6680 端口，如需修改：

编辑 `rsbuild.config.ts`:
```typescript
export default defineConfig({
  server: {
    port: 3000  // 修改为其他端口
  }
})
```

## 发布出包流程（CI/CD）

自动出包由两个 GitHub Actions workflow 接力完成，需要提前配置一个 Secret。

### 需要配置的 Secret

| Secret 名 | 用在哪 | 是否必须 | 说明 |
|-----------|--------|---------|------|
| `CI_TOKEN` | `create-release-tag.yml` | **必须** | PAT（个人访问令牌），用于推 tag 并触发下游构建 |
| `GITHUB_TOKEN` | `release.yml` | 自动提供，无需配置 | GitHub 自带 |
| `TAURI_PRIVATE_KEY` | `release.yml` | 可选 | Tauri 自动更新器签名私钥，不做自动更新可留空 |
| `TAURI_KEY_PASSWORD` | `release.yml` | 可选 | 上面私钥的密码 |

**为什么必须用 `CI_TOKEN` 而非默认 `GITHUB_TOKEN`：** GitHub 有防递归机制，用默认 `GITHUB_TOKEN` 推的 tag 不会再触发其他 workflow。只有用 PAT（`CI_TOKEN`）推的 tag 才能触发 `release.yml` 自动构建。

**配置方法：**
1. 生成 PAT：GitHub → Settings → Developer settings → Personal access tokens
   - Classic：勾选 `repo` scope
   - Fine-grained：给目标仓库 **Contents: Read and write**
2. 存入仓库：仓库 → Settings → Secrets and variables → Actions → New repository secret，名字填 `CI_TOKEN`

### 全流程

```
本地                          GitHub Actions #1              GitHub Actions #2
──────────────────────────────────────────────────────────────────────────
pnpm bump:version   ──push──▶  create-release-tag.yml  ──tag──▶  release.yml
（改版本+提交）                （打 tag）                        （多平台构建出包）
```

**第 1 段：本地 `pnpm bump:version`**（`scripts/bump-version.js`）
1. `pnpm version <mode> --no-git-tag-version` → 只改 `package.json` 版本号，不打 tag
2. `git commit -m "chore(release): v<新版本>"` → 生成特定格式的提交
3. ⚠️ 脚本只 commit **不 push**，需手动 `git push` 到 `main`

**第 2 段：`create-release-tag.yml`**（监听 push 到 main）
1. 读取最新 commit message
2. 只有以 `chore(release):` 开头才继续，普通提交跳过
3. 从 `package.json` 取版本号
4. 调 GitHub API 查 `v<版本>` tag 是否已存在（防重复）
5. 不存在 → 用 `CI_TOKEN` 创建并推送 tag `v<版本>`

**第 3 段：`release.yml`**（监听 push tag `v*`）
1. matrix 三平台并行构建：macOS Intel / macOS M1(ARM) / Windows
2. 下载 ffmpeg sidecar 二进制放进 `src-tauri/binaries`
3. `tauri-apps/tauri-action` 构建各平台包（app/dmg/msi）
4. 上传到一个**草稿(draft) Release**

**最后一步（手动）：** 构建完 Release 是草稿状态，需去 GitHub Releases 页面手动 **Publish** 才对外可见。

### 一句话总结

> `pnpm bump:version` → `git push` →（等 CI 跑完）→ 去 Releases 点 **Publish**。中间打 tag、三平台构建、传产物全自动。

### ⚠️ 注意事项

- **macOS 包未签名未公证**：`tauri.conf.json` 中 `signingIdentity` 和 `entitlements` 均为 `null`，自动出的 macOS 包未做代码签名和公证。用户首次打开需右键→打开，且未来较新的 macOS 会有 Gatekeeper 相关提示。若需消除，需配置 `signingIdentity` + notarization（`APPLE_ID` / `APPLE_TEAM_ID` 等）。
- **本地 `pnpm app:build`**（裸 `tauri build` 不带 `--target`）只产出**当前机器架构**的原生包（如在 Apple Silicon 上打出的是 ARM64 版）。打指定架构需用 `pnpm app:build:multi --target macos-intel|macos-arm64|windows-x64`。
